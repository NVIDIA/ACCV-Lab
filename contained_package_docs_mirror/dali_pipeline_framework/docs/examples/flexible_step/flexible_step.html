Flexible Image Decoding Step

See also

The implementation of ImageDecoder is the one used in the core DALI pipeline framework package. It can found in the repository at packages/dali_pipeline_framework/accvlab/dali_pipeline_framework/processing_steps/image_decoder.py.

Next, we will have a look at the flexible image decoding step. The functionality is similar to the Simple Image Decoding Step version, but the implementation does not make any assumptions about how many images need to be processed or where in the input data they are located. Instead, it makes use of find_all_occurrences() to automatically discover all images in the input data by name.

Important

While the discovery of the images requires additional steps in the implementation, note that these steps are performed at DALI graph construction time (as no DataNode data or DALI operators are involved, only “normal” Python data). Therefore, even if elaborate discovery is needed, there will be no runtime cost during pipeline execution.

For example, if the input data to this flexible image decoding step is in the format as expected by the Simple Image Decoding Step version, the resulting DALI graph will be the same for both versions, and they will therefore run in the same way.

 15try:
 16    from typing import override
 17except ImportError:
 18    from typing_extensions import override
 19
 20import nvidia.dali.types as types
 21import nvidia.dali.fn as fn
 22
 23# @NOTE: Import the needed functionality from the DALI pipeline framework package. Here, we are inside the
 24# `accvlab.dali_pipeline_framework` package, and so, in contrast to the simple step example, we use relative
 25# imports.
 26from .pipeline_step_base import PipelineStepBase
 27from ..pipeline.sample_data_group import SampleDataGroup
 28
 29
 30class ImageDecoder(PipelineStepBase):
 31    '''Decode images.
 32
 33    Behavior:
 34      - Finds all images by name, decodes them (to RGB or BGR), and replaces the encoded image data
 35        by the decoded version in place.
 36      - Image search happens at DALI graph construction time; only the actual decoding operator is part of
 37        the DALI graph. This means that the runtime performance is not affected by the search for images.
 38    '''
 39
 40    def __init__(
 41        self,
 42        image_name: str,
 43        use_device_mixed: bool,
 44        hw_decoder_load: float = 0.65,
 45        as_bgr: bool = False,
 46    ):
 47        '''
 48
 49        Args:
 50            image_name: Name of the image data field(s) to decode
 51            use_device_mixed: If ``True``, decoding will be partially performed on the GPU and the resulting
 52                images will be located in GPU memory. If ``False``, only the CPU is used.
 53            hw_decoder_load: In case of ``use_device_mixed==True``, this parameter sets the fraction of the
 54                workload to be performed by decoding hardware (as opposed to software CUDA kernels).
 55            as_bgr: Whether to output BGR images (instead of RGB images).
 56        '''
 57
 58        self._image_name = image_name
 59        self._use_device_mixed = use_device_mixed
 60        self._hw_decoder_load = hw_decoder_load
 61        self._as_bgr = as_bgr
 62
 63    @override
 64    def _process(self, data: SampleDataGroup) -> SampleDataGroup:
 65        # @NOTE
 66        # We need to override the main processing step function and implement our functionality here.
 67        #
 68        # Note that functionality related to getting the image data fields (e.g.
 69        # `data.find_all_occurrences()`, `data.get_item_in_path()`, ...)
 70        # is performed at DALI graph construction time, and therefore is free in terms of runtime performance
 71        # when running the pipeline.
 72
 73        # @NOTE
 74        # In contrast to the simple example (see
 75        # `packages/dali_pipeline_framework/examples/simple_processing_step_example/simple_example_image_decoder.py`),
 76        # we search for all images to process (same name, but may be in different locations in the input data,
 77        # e.g. one image per camera). Note that if more than one image name is needed, multiple `ImageDecoder`
 78        # steps can be used in the pipeline, one per name.
 79        image_paths = data.find_all_occurrences(self._image_name)
 80
 81        # @NOTE
 82        # We loop over all images found. Note that while in the DALI graph, loops are not supported,
 83        # we can still use Python loops which do not depend on DALI-specific data, and are therefore executed
 84        # at graph construction time. Such loops are effectively unrolled in the DALI graph.
 85
 86        # @NOTE: For each image found:
 87        for ip in image_paths:
 88            # @NOTE: Get the image to process. We use the `get_item_in_path()` method to get the image data at
 89            # the path `ip` without having to traverse the data structure manually.
 90            encoded_image = data.get_item_in_path(ip)
 91
 92            # @NOTE
 93            # Decode the image (as in the simple example, but with the additional configuration parameter
 94            # `as_bgr` for improved flexibility)
 95            decoding_output_type = types.RGB if not self._as_bgr else types.BGR
 96            image = fn.decoders.image(
 97                encoded_image,
 98                device="mixed" if self._use_device_mixed else "cpu",
 99                hw_decoder_load=self._hw_decoder_load,
100                output_type=decoding_output_type,
101            )
102
103            # @NOTE
104            # Set the image data to the decoded version. Here, we use the `set_item_in_path()` method to set
105            # the image data at the path `ip` without having to traverse the data structure manually.
106            data.set_item_in_path(ip, image)
107
108        # @NOTE: Return the resulting data structure with the decoded images.
109        return data
110
111    @override
112    def _check_and_adjust_data_format_input_to_output(self, data_empty: SampleDataGroup) -> SampleDataGroup:
113        # @NOTE
114        # As in the simple example, we need to override and implement this function to adjust the data format
115        # to the output data format & check for compatibility.
116
117        # @NOTE
118        # Here, we do not assume a specific image location (see notes in `_process()` above). Instead, we
119        # enforce that at least one image can be found.
120        image_paths = data_empty.find_all_occurrences(self._image_name)
121        if len(image_paths) == 0:
122            raise KeyError(
123                f"No occurrences of images found. Fields containing images are expected to have the name "
124                f"'{self._image_name}', as specified in the constructor."
125            )
126
127        # @NOTE: For each image found:
128        for ip in image_paths:
129            # @NOTE: We check the format of the image.
130            image_parent = data_empty.get_parent_of_path(ip)
131            if image_parent.get_type_of_field("image") != types.DALIDataType.UINT8:
132                raise ValueError(f"Image data at path `{ip}` in the input data is not of type UINT8")
133
134        # @NOTE
135        # Return the output data format (which is the same as the input data format, as we did not add or
136        # adjust any data fields).
137        return data_empty
